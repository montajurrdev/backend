// config and .env issue:
//
// .env file store all secret keys. need to add it to gitignore
// dotenv pkg => configure process.env
// dotenv reads .env and parse key=value pairs
// injects all environment variables into process.env

// we can access anywhere into node.js runtime

//
// config/ folder => normal project organization
// for keeping configuration file => db.js, or others file

// config pkg => special configuration library
                // it configure application by environment
                // inside config/ folder default.json, development.json, production.json
                // config pkg reads JSON inside config by environment
                // provides values through config.get()


// for small/medium project => we can only use:
// .env + process.env  for secret keys


// or we can use: .env + process.env + config pkg
// for configure application by environment
// this is best setup
// but best for large/enterprise project

// only config pkg harder to deploy application
// it can't hide secret
// if we add config, or config/files inside gitignore. hard to deploy

// environment variables: two types based on where they are stored:
                .env variables   → stay at   project file
                system environment variables → stay at operating system

access through:
.env variables   => process.env (dotenv)
system variables => export V_N=value


// environment variables can be classified in multiple ways:

                       export NODE_ENV=development

Local Shell variables → NAME=John - only current shell can access it.child programs cannot access it
