// Proper HTTP status codes

400 = Bad Request  => the user sent data invalid (wrong email without domain, wrong password or incomplete)
401 = Unauthorized => user is not logged in or token invalid — authentication failed (email or password invalid)
403 = Forbidden => user logged in but doesn't have permission (only admin can delete)
404 = Not Found => requested resource doesn't exist (user/product not found) 
409 = Conflict => data already exists or conflict with current data (email already exists/ username already taken)
500 = Internal Server Error => something broke on the Server (db crash) — we don't handle it manually, it handle global middleware

// Global error middleware

our current errors are handling inside try/catch.
    res.status(err.statusCode).json({
      status: err.status,
      message: err.message, 
    })

we will have to write this error handler again and again
in: 
authController, productController, ownerController, shopController
that's duplicated issue


we will handle it through global Error middleware

src/
├── middlewares/
│   └── globalErrorHandler.js


# after it
whole project error will handles within one handler/middleware

# we don't need to handle manually
try {}
catch(err){
   next(err)
}
# next(err) forwards error to globalErrorHandler


# global middleware sit after all routes/ always last
